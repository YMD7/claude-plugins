# .claude/terse-mode/custom.md — プロジェクト固有ルール

terse-mode スキルのプロジェクト固有拡張を定義するファイル。配置されている場合、コア SKILL.md の全ルールより custom.md が優先されます。

## 配置場所

```
<project-root>/.claude/terse-mode/custom.md
```

## 代表的なユースケース

### 1. 自動化システム連携

パターンマッチ型の自動承認ボット（番頭 bot・CI ゲート自動承認等）と連携する場合、特定フレーズの文言維持を指定する。

````markdown
## 最優先ルール: 確認テンプレート文言厳守

- 確認プロンプトの文言は一字一句維持する（助詞省略禁止）
- 例: 「ワークツリーを作成しましょか？」を「ワークツリー作成しましょか？」と短縮してはならない
- 違反するとパターンマッチが失敗し、自動承認が機能しない
````

### 2. セマンティックフラグの定義

メッセージ末尾に付与するフラグと自動化システムの動作を記述する例:

````markdown
| フラグ       | 用途                    | システムの動作       |
| ------------ | ----------------------- | -------------------- |
| `[ASK]`      | 定型確認（y/n、選択肢） | 自動承認対象         |
| `[COMPLETE]` | 作業完了報告            | 即時通知             |
| `[DIALOG]`   | 非定型対話              | アイドル後通知       |
````

### 3. 口調指定

````markdown
## 口調

- 通常モード（圧縮OFF）では大阪弁で応答する
- 一人称は「ワイ」
````

### 4. ワークフロー系スキル名の具体列挙

````markdown
- ワークフロー系スキル: `/create-worktree`, `/plan-task`, `/pr`, `/code-review`, `/merge-pr` 等
````

## 最小例

```markdown
# terse-mode custom

## 口調

- 通常モードでは敬体で応答する

## 追加禁止事項

- コミットメッセージを圧縮しない（コア側と同じだが明示）
```

## 優先順位

custom.md の記述はコア SKILL.md の全ルールに優先します。矛盾があれば custom.md に従います。
